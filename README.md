Телеграм-бот для автоматизации планирования игр в масштабных D&D кампаниях. Бот помогает координировать расписание нескольких групп, учитывает часовые пояса и автоматически напоминает о сессиях.

## 🚀 Основной функционал

### Для Администратора (Мастера):
*   **Управление группами:** Создание и настройка составов игроков.
*   **Запуск планирования:** Команда на сбор свободного времени для конкретной группы на 2 недели вперед.
*   **Подтверждение:** Утверждение предложенного ботом оптимального времени.

### Для Игроков:
*   **Выбор времени:** Удобный интерфейс с календарем и почасовым выбором (12:00 - 24:00).
*   **Личные настройки:** Установка часового пояса и постоянного цикличного расписания на неделю.
*   **Управление записями:** Просмотр и удаление своих слотов доступности.

### Умная логика:
*   **Алгоритм пересечений:** Поиск идеального окна для всех участников.
*   **Система рекомендаций:** Если пересечений нет, бот предлагает варианты с минимальными корректировками для игроков.
*   **Авто-напоминания:** Каждые 3 часа для тех, кто не заполнил время, и уведомления за 5/1 час до игры.
*   **Цикличность:** Автоматический переход очереди планирования к следующей группе после завершения игры.

## 🛠 Технологический стек
*   **.NET 8 (Console Application)**
*   **Telegram.Bot SDK** — взаимодействие с API.
*   **Entity Framework Core** — работа с базой данных (SQLite/PostgreSQL).
*   **Background Tasks** — фоновые службы для уведомлений и таймеров.

## 📂 Структура проекта
```text
/Data                 # Модели БД и контекст (Entities, AppDbContext)

```

## ⚙️ Установка и запуск

1. Клонируйте репозиторий.
2. Создайте файл `appsettings.json` в корне проекта:
   ```json
   {
   "BotToken": "ВАШ_ТОКЕН_ОТ_BOTFATHER",
   "DbConnectionString": "Data Source=dnd_planner.db"
   }
   ```
3. Выполните миграции БД: `dotnet ef database update`.
4. Запустите проект: `dotnet run`.

## 🔄 Логика работы планировщика
1. Мастер запускает цикл: `/start_voting [group_id]`.
2. Бот собирает данные, напоминая игрокам о заполнении.
3. Если за 2 цикла время не найдено, бот переключается на следующую группу.
4. При отказе более 1 игрока в день партии, сессия автоматически отменяется и поиск времени возоновляется.
